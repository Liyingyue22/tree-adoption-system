# 项目协作文档

## 一、项目名称
深汕天子山树木认养系统

## 二、项目简介
本项目为《管理信息系统开发方法》课程期末项目，围绕“树木认养”业务场景开发一个前后端分离系统，支持：

- 用户登录注册
- 认养大厅查看树木
- 树木详情查看
- 下单认养
- 我的树与摄像头抓拍 / 视频查看
- 管理员后台管理树木、订单、公司、养护记录、摄像头状态

---

## 三、仓库用途
本仓库用于统一管理：

- 前端代码
- 后端代码
- 数据库资料
- 接口文档
- 联调记录
- 协作说明
- 答辩素材

避免成员之间重复传文件、版本混乱。

---

## 四、当前仓库结构

## 七、项目启动方式

### 1. 后端 B 启动
```bash
cd backend-b
pip install -r requirements.txt
python app.py
```

接口地址：
```text
http://localhost:8080/api
```

---

### 2. 用户端启动
```bash
cd frontend-user
npm install
npm run dev
```

---

### 3. 管理端启动
```bash
cd frontend-admin
npm install
npm run dev
```

## 十二、协作约定

1. 代码统一提交到本仓库，不再互相单独发最终版代码
2. 用户端只改 `frontend-user/`
3. 管理端只改 `frontend-admin/`
4. 后端接口修改需同步更新文档与联调记录
5. 数据库内容若需调整，优先同步 DBA / 后端
6. 遇到问题先判断归属：
   - 前端问题
   - 后端问题
   - 数据库问题
   - 硬件 / 摄像头问题

---
